# 企业内容包模板使用说明

> 适用工程：`apps/card-web`  
> 完整示例：`src/tenants/template/tenant.ts`  
> 通用模板入口：根地址、`/c/template`  
> 首个企业 seed：`/c/tuotu`

## 1. 使用原则

`template` 是可运行的无品牌通用模板，用来说明企业内容包的完整结构。它不是任何企业的官网，也不包含可直接发布的企业事实。

接入新企业只允许修改三类内容：

1. 新企业的 `src/tenants/{tenant}/tenant.ts` 内容包。
2. 新企业的 `public/tenants/{tenant}/assets/` 素材。
3. `src/tenants/index.ts` 中的 import、注册表记录和 export。

不要修改 `App.tsx` 或 `styles.css`。如果现有 7 类 block 不能准确表达企业内容，应先记录需求并评估通用类型，而不是在页面中加入企业名称判断或复制一套页面。

## 2. 入口与文件

当前入口：

| 地址 | 内容 |
|---|---|
| `/` | 无品牌通用模板 |
| `/c/template` | 无品牌通用模板的显式地址 |
| `/c/tuotu` | 首个企业 seed |
| `/?tenant=tuotu` | 查询参数形式的首个企业 seed |
| `/c/{unknown}` | 无品牌的 tenant 不存在状态 |

模板相关文件：

```text
apps/card-web/
  src/
    domain/card.ts
    tenants/
      index.ts
      defineTenant.ts
      template/
        tenant.ts
  public/
    tenants/
      template/
        assets/
```

- `domain/card.ts` 定义内容包和 7 类 block 的 TypeScript 类型。
- `tenants/defineTenant.ts` 提供 `defineTenant()` 类型约束。
- `tenants/template/tenant.ts` 是新企业应复制的完整内容包示例。
- `tenants/index.ts` 是静态 tenant 注册表和路径解析入口。
- `public/tenants/template/assets/` 只包含模板示意素材，不可直接作为企业素材上线。

## 3. 内容包顶层最小字段

一个可运行内容包必须提供以下顶层字段：

| 字段 | 最小要求 |
|---|---|
| `id` | tenant key，小写字母或数字开头，仅允许小写字母、数字、下划线和短横线，最长 63 个字符 |
| `version` | 非空版本字符串 |
| `seo` | `title`、`description` |
| `brand` | 品牌文案、Logo、首页无障碍标签和官网行动 |
| `theme` | 模式、行动颜色、深浅主题令牌、首屏遮罩和圆角令牌 |
| `hero` | 首屏文案、图片、行动与指标数组 |
| `sections` | 1 至 12 个 block |
| `assistant` | 助手文案、至少 1 条知识、快捷问题、标签和拒答内容 |
| `footer` | 品牌说明、免责声明和返回顶部行动 |

### 3.1 `brand`

必填字段：

```text
name
shortName
tagline
headerDescriptor
logo.src
logo.alt
homeAriaLabel
officialAction.kind
officialAction.label
officialAction.target
```

`officialAction.kind` 必须是 `external`。`target` 只能使用有效的 `http`、`https`、`mailto` 或 `tel` 地址。没有已确认官网或联系地址时，不要保留模板的 `https://example.com`，应先补齐有效入口再预览发布。

### 3.2 `theme`

必填字段：

- `defaultMode`：`system`、`light` 或 `dark`。
- `action`、`onAction`。
- `light` 与 `dark`：分别提供 `accent`、`accentStrong`、`accentSoft`、`background`、`surface`、`surfaceRaised`、`surfaceMuted`、`text`、`textSoft`、`textFaint`、`line`、`lineStrong`、`shadow`。
- `heroOverlay.light`、`heroOverlay.dark`。
- `radiusCard`、`radiusControl`、`radiusSmall`。

模板已经给出完整字段。接入时只替换令牌值，不修改通用 CSS。

### 3.3 `hero`

必填字段：

```text
id
kicker
titleLines
summary
art
actions
metrics
```

约束：

- `titleLines` 包含 1 至 2 行非空标题。
- `art` 至少包含非空 `src` 和字符串 `alt`。
- `actions` 必须是数组，最多 2 个行动。
- `metrics` 必须是数组，最多 4 项。每项包含 `value`、`label` 和 `note`。

企业没有可核验指标时，可以使用空数组，不能沿用模板数字或虚构业绩。

### 3.4 `assistant`

必填内容包括助手标题、状态、副标题、入口标签、初始消息、交互标签、免责声明、知识库、快捷问题和拒答内容。

每条 `knowledgeBase` 知识需要：

```text
id
question
shortQuestion
answer
keywords
source
```

约束：

- 知识库至少 1 条。
- 每条知识 ID 非空且唯一。
- `keywords` 包含 1 至 20 个非空关键词。
- `quickQuestionIds` 只能引用已有知识 ID。
- FAQ block 的 `itemIds` 也只能引用已有知识 ID。

模板助手只解释资料接入和审核流程。新企业必须全部替换为该企业已确认的问答、来源和拒答边界。

## 4. 7 类 block 与最小必填字段

所有 block 都需要以下公共字段：

| 字段 | 要求 |
|---|---|
| `type` | 下表中的一种固定类型 |
| `id` | 非空且在当前内容包中唯一 |
| `navLabel` | 非空导航短标签，即使 `showInNav` 为 `false` 也要提供 |
| `showInNav` | 布尔值，整个页面最多 6 个 block 可设为 `true` |
| `heading` | 非空主标题 |
| `description` | 字符串，可按设计需要使用空字符串 |
| `eyebrow` | 可选，提供时必须非空 |

### 4.1 `feature-grid`

用途：展示业务、产品、能力或受众分组。

附加字段：`businesses`，包含 2 至 6 项。每项需要：

```text
icon
eyebrow
title
description
status
points
```

`points` 包含 2 至 6 条非空文字。`icon` 使用当前通用图标 token，例如 `buildings`、`globe`、`book`、`path`、`code`、`student` 或 `users`。

### 4.2 `media-showcase`

用途：用一张主视觉配合产品、服务或场景能力说明。

附加字段：

```text
capabilities
action
visualLabel
visualTitle
visual
```

`capabilities` 包含 2 至 6 项，每项需要 `icon`、`title` 和 `description`。`action` 需要 `kind`、`label` 和 `target`。`visual` 至少需要 `src` 和字符串 `alt`。

### 4.3 `process`

用途：展示工作流程、成长路径和参与角色。

附加字段：

```text
steps
audienceHeading
audiences
```

`steps` 包含 2 至 6 项，每项需要 `title` 和 `text`。`audiences` 包含 2 至 6 项，每项需要 `icon`、`title` 和 `description`。

### 4.4 `evidence`

用途：展示已获授权的案例、指标、资质、主题或支持信息。

附加字段：

```text
visual
headlineMetric
metricDescription
themesAriaLabel
themes
caveat
supportHeading
supportNote
supportNames
```

`themes` 与 `supportNames` 各包含 2 至 6 条非空文字。没有已获授权证据时，应移除该 block，不要沿用模板审核示意或其他企业数据。

### 4.5 `engagement`

用途：展示合作、咨询、接入或转化路径。

附加字段：

```text
steps
cta.title
cta.description
cta.action
```

`steps` 包含 2 至 6 项，每项需要 `title` 和 `text`。`cta.action` 需要有效的 `kind`、`label` 和 `target`。

### 4.6 `faq`

用途：从企业知识库选择页面常见问题。

附加字段：`itemIds`，包含 2 至 6 个已有知识 ID。`action` 可选，提供时必须是有效行动。

不要直接复制模板 FAQ。即使问题形式相似，答案、来源和边界也必须改为该企业已审核信息。

### 4.7 `closing`

用途：收束页面并提供最后行动入口。

附加字段：

```text
art
actions
```

`art` 至少需要 `src` 和字符串 `alt`。`actions` 必须是数组，最多 2 个行动。

## 5. 行动与素材格式

### 5.1 行动

支持三种行动：

| `kind` | `target` 格式 | 用途 |
|---|---|---|
| `anchor` | `#section-id` | 页面内跳转 |
| `external` | 有效的 `http`、`https`、`mailto` 或 `tel` 地址 | 官网、电话、邮件等外部入口 |
| `assistant` | 最长 160 个字符的问题或 `open` | 打开助手或提交预设问题 |

每个行动必须有非空 `label`。行动组最多 2 个。

### 5.2 素材

素材对象至少提供：

```text
src
alt
```

`caption`、`width` 和 `height` 可选。提供尺寸时必须是正整数。装饰图的 `alt` 可以是空字符串，有信息含义的图片必须写清楚替代文本。

新企业素材统一存放在：

```text
public/tenants/{tenant}/assets/
```

内容包中的素材函数也要同步改为该 tenant 路径。不要引用 `template` 或 `tuotu` 目录中的图片。

## 6. 复制接入步骤

以下以 tenant key `your-company` 为例。它只是路径示意，不是仓库内已存在的企业。

### 步骤 1：确认资料与 key

完成企业主体、品牌、业务、证据、联系入口、FAQ 和素材授权收集。确定稳定的 tenant key，例如 `your-company`。

### 步骤 2：复制内容包示例

复制：

```text
src/tenants/template/tenant.ts
```

到：

```text
src/tenants/your-company/tenant.ts
```

把导出名 `templateTenant` 改为 `yourCompanyTenant`，把 `id` 改为 `your-company`，把素材路径中的 `template` 改为 `your-company`。

### 步骤 3：复制并替换素材

建立：

```text
public/tenants/your-company/assets/
```

替换 Logo、首屏图和各 block 图片。每个素材都要确认版权、品牌、肖像和公开范围。

### 步骤 4：替换内容包

依次替换：

1. `seo` 与 `brand`。
2. `theme`。
3. `hero` 文案、行动、图片和指标。
4. `sections` 中需要保留的 block。
5. `assistant.knowledgeBase`、快捷问题和拒答内容。
6. `footer` 说明与免责声明。

企业不需要的 block 直接从 `sections` 数组中移除。不要为了保持 7 个 block 而编造内容。

### 步骤 5：增加注册记录

在 `src/tenants/index.ts` 中完成三处修改：

1. import `yourCompanyTenant`。
2. 在 `tenantRegistry` 增加 `"your-company": yourCompanyTenant`。
3. export `yourCompanyTenant`。

不要改 tenant 解析逻辑。

### 步骤 6：运行验证

在仓库根目录执行：

```powershell
pnpm web:test
pnpm web:build
```

启动后访问 `/c/your-company`，至少检查：

- 页面标题、Logo、品牌、颜色和所有图片均属于当前企业。
- 导航锚点、外链、电话、邮件和助手行动有效。
- FAQ ID、快捷问题和来源正确。
- 未出现 `template`、`tuotu`、`example.com` 或其他企业内容。
- 桌面端与移动端可完整浏览。
- 所有数据、案例、合作方和联系信息已经企业审核。

## 7. 模板占位内容禁用清单

以下内容只为解释模板结构，不得直接上线：

- “企业数智名片通用模板”“通用模板”等模板品牌文案。
- `https://example.com`。
- `template-mark.webp`、`template-hero.webp`、`template-content-kit.webp`、`template-review.webp`。
- “等待企业资料”“等待业务确认”“等待证据审核”“等待渠道配置”等状态。
- “占位”“示意”“非真实企业页面”等模板提示。
- “4 类资料”“7 类模块”“5 条问答”等模板说明数字。
- 模板知识库中的资料接入问答和“通用模板说明”来源。
- 模板页脚、免责声明和资料助手名称。

如果一项企业事实尚未确认，可以保留在内部资料台账中，但不能用模板占位值替代后上线。对应 block 应从 `sections` 中移除，或继续作为内部预览，不得对外发布。

## 8. 发布前检查

- [ ] tenant key 与目录、内容包 `id`、注册表 key 完全一致。
- [ ] 模板导出名、素材路径、SEO 和品牌文案已全部替换。
- [ ] 没有修改 `App.tsx` 或 `styles.css`。
- [ ] 页面只使用当前企业素材目录。
- [ ] 7 类 block 仅保留有真实内容的类型。
- [ ] 所有列表数量和行动格式通过校验。
- [ ] FAQ 与快捷问题只引用当前企业知识 ID。
- [ ] 模板占位文案、示意数字和 `example.com` 已清除。
- [ ] 主体、数据、案例、合作方、素材和联系方式已经审核。
- [ ] `pnpm web:test` 与 `pnpm web:build` 通过。

模板的目的，是让企业接入只发生在内容层。只要新增企业需要修改通用页面结构，就应先判断这是可复用产品能力还是单一企业特例。
