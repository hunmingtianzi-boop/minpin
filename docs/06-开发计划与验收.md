# 06 开发计划、任务拆分与验收

版本：V1.0  
排期单位：相对工作日；确认团队与外部资源后再写日历日期

## 1. 排期假设

以下 3-4 周 MVP 仅在至少具备这些有效产能时成立：产品/项目 1、前端 1-2、后端/AI 2、测试 1（可兼运维）、设计支持 0.5。若核心开发仅 1-2 人，应按产能重排，不能保留原周期同时保留全部 P0。

外部前置：样板企业资料、域名/部署主体、模型与云资源账号、隐私政策责任人必须及时可用。等待外部资料不计为研发“已完成”。

## 2. 里程碑

| 里程碑 | 结果 | 建议相对周期 | 进入条件 |
|---|---|---|---|
| M0 开发就绪 | 范围、契约、环境、样板数据、评测集可用 | 2-3 天 | P0 待确认项有责任人和截止日 |
| M1 V0.1 Demo | 1 家企业的展示→问答→纪要纵向切片 | 第 1 周末 | M0 通过 |
| M2 V1.0 MVP | 真实小规模企业试用，闭环、安全、运维可验收 | 第 3-4 周 | M1 问题关闭、数据/合规材料齐备 |
| M3 V1.5 试点 | 批量开通、商会聚合、导出和运营支撑 | MVP 后 4-8 周 | 试点名单、运营人、MVP 指标通过 |
| M4 V2 商业化 | 套餐、组织、外部集成、高级检索 | 由试点数据立项 | 商业化门槛通过 |

原需求中的 2026 年 7 月中 MVP 与 8 月 1000 用户目标没有团队产能、试点资源和合规前置支撑，本基线将其视为期望日期，不作为未经重新估算的工程承诺。

## 3. Sprint 0：开发就绪（2-3 天）

| ID | 任务 | 交付物 | Owner 类型 |
|---|---|---|---|
| S0-01 | 冻结 MVP 和非目标 | 需求基线签字/评审记录 | 产品/甲方 |
| S0-02 | 关闭技术选型 | ADR、锁定运行时和 Provider 接口 | 技术负责人 |
| S0-03 | 初始化 Monorepo/CI | lint、test、build、migration 流水线 | 研发/DevOps |
| S0-04 | 本地基础设施 | PostgreSQL/pgvector、Redis、MinIO、健康检查 | 后端 |
| S0-05 | 建立身份与租户骨架 | migration、request scope、RLS 冒烟测试 | 后端/安全 |
| S0-06 | 建立 OpenAPI 与生成客户端 | 契约基线、前端调用层 | 前后端 |
| S0-07 | 准备样板企业数据 | 3 产品、3 案例、20 FAQ、禁答清单 | 产品/甲方 |
| S0-08 | 建立 AI 评测集 | 20 标准题 + 跨租户/注入集 | AI/测试 |
| S0-09 | 合规前置 | AI 标识、隐私提示、同意文案待审稿 | 产品/法务 |

退出标准：新成员可按文档在 30 分钟内启动依赖；CI 全绿；A/B 企业 RLS 冒烟通过；样板数据和模型账号可用。

## 4. Sprint 1：名片与企业内容纵向切片（约 5 天）

| ID | 任务 | 验收 |
|---|---|---|
| S1-01 | 登录、刷新、退出、membership 权限 | Token 可撤销；无权限资源不泄露 |
| S1-02 | 企业资料、产品、案例、FAQ、禁答 CRUD | 状态和乐观锁可用；跨企业测试通过 |
| S1-03 | 名片 CRUD、随机 slug、启停 | 停用后公开访问立即失效 |
| S1-04 | 访客 H5 首页、产品/案例详情 | 微信内置浏览器和主流移动端可用 |
| S1-05 | 访问会话与基础事件 | PV/UV 可从原始记录抽样对账 |
| S1-06 | 基础企业后台 | 可维护 Demo 所需内容和名片 |
| S1-07 | 发布→Outbox→索引任务骨架 | 任务幂等、失败可见、旧版本不丢 |

退出标准（V0.1 展示部分）：可创建 1 家样板企业和名片，扫码/链接进入 H5，后台修改发布内容后无需重新部署。

## 5. Sprint 2：RAG 对话与证据链（约 5 天）

| ID | 任务 | 验收 |
|---|---|---|
| S2-01 | 业务对象切片与 embedding | 内容 hash 幂等；索引状态可追踪 |
| S2-02 | company/card/visibility 过滤检索 | A 企业问题永不命中 B 企业片段 |
| S2-03 | 对话/消息/AI run/引用模型 | 每次回答可关联模型、Prompt、引用和耗时 |
| S2-04 | SSE 流式回答 | 断线、超时、重复提交行为可预测 |
| S2-05 | 意图、证据门控与拒答 | 无依据/报价/合同/越权按策略处理 |
| S2-06 | 输入输出安全与限流 | 基础注入测试和公共接口滥用测试通过 |
| S2-07 | AI 对话 UI | 明确 AI 身份、等待/错误/重试/引用状态 |
| S2-08 | 离线评测命令与报告 | 固定数据集可重复运行并保存版本 |

退出标准（V0.1 完整 Demo）：样板企业可完成展示→AI 问答→保存对话→生成一次纪要演示；不存在已知跨企业泄露。

## 6. Sprint 3：留资、线索、纪要与主人后台（约 5 天）

| ID | 任务 | 验收 |
|---|---|---|
| S3-01 | 分层同意记录和隐私提示 | chat/lead consent 可审计；未同意不得留资 |
| S3-02 | 加密访客 Profile 与留资 | 列表脱敏；授权详情读取有审计 |
| S3-03 | 线索与状态机 | new/viewed/following/won/lost/invalid 合法流转 |
| S3-04 | 跟进记录 | 追加式历史、操作者和时间完整 |
| S3-05 | 幂等纪要任务 | 关闭/超时触发；新消息使旧纪要 stale |
| S3-06 | 主人看板和列表 | 主人只看本人名片数据，企业管理员看本企业 |
| S3-07 | 站内通知 | 新线索/纪要可见；不依赖外部推送 |
| S3-08 | 隐私申请流程骨架 | 请求可登记、核验、处理和留证 |

退出标准：真实访客可以自愿留资；主人能查看完整证据链、纪要和线索并跟进；PII 不出现在普通日志。

## 7. Sprint 4：知识进化与试点硬化（约 5 天）

| ID | 任务 | 验收 |
|---|---|---|
| S4-01 | 知识缺口识别与去重 | 无依据问题可归集，不把敏感拒答当知识 |
| S4-02 | AI 草稿与人工审核 | AI 草稿不能绕过审核发布 |
| S4-03 | 新版本重索引与原子切换 | 失败保留旧版；成功后原问题回归 |
| S4-04 | 审计日志和 break-glass | 敏感查看/导出/审核/删除均可追踪 |
| S4-05 | 备份、恢复、清理任务 | 完成一次恢复演练；失败有告警 |
| S4-06 | 性能、兼容和错误降级 | 达到非功能门槛；第三方故障不破坏基础页面 |
| S4-07 | 安全回归 | RLS、IDOR、注入、速率、文件/输入边界通过 |
| S4-08 | 运营数据对账 | PV/UV/对话/留资/纪要口径可抽样复核 |
| S4-09 | UAT 与问题清零 | P0 缺陷为 0，P1 有书面豁免/计划 |

退出标准（V1.0 MVP）：[第一阶段验收清单](#10-v10-mvp-验收清单)全部有证据，部署/回滚/恢复可执行，法务与业务上线门槛已签署。

## 8. V1.5 试点 Backlog

- 企业与员工批量导入、失败回执和重复处理。
- 企业资料提交进度、催办与运营清单。
- 商会聚合看板；明确最小样本阈值，防止反推单一企业数据。
- 后台显示回答来源、评测集管理和质量趋势。
- 访客/线索/对话异步导出和审批审计。
- 外部消息 Provider：优先选择一个真实可用渠道，具备授权、幂等、重试和送达状态。
- 生产容量、成本预算、模型配额和供应商切换演练。
- 试点培训、资料运营、问题响应和周报机制。

## 9. Definition of Done

每个开发任务只有同时满足以下条件才能完成：

- 需求和异常路径有明确验收；UI 包含 loading/empty/error/permission 状态。
- OpenAPI/事件 Schema 与实现一致，兼容性变更已评审。
- migration 可正向执行；破坏性变更有回滚/向前修复方案。
- 单元/集成/契约测试通过，关键路径有 E2E；租户数据路径有 A/B 越权用例。
- 日志、指标、trace、错误码和告警点已补齐，日志无秘密和明文 PII。
- 幂等、重试、超时和并发行为已验证。
- 代码审查完成；无高危依赖/静态扫描问题，无硬编码密钥。
- 文档、ADR、运行手册和已知限制同步更新。
- 产品/测试提供可复现的验收证据，不以“本机看起来正常”作为完成。

## 10. V1.0 MVP 验收清单

### 10.1 功能

- [ ] 平台可创建企业并开通企业管理员。
- [ ] 企业可录入、审核、发布企业资料、产品、案例、FAQ 和禁答规则。
- [ ] 企业可创建、发布、停用至少一张名片。
- [ ] 访客可通过二维码/链接在微信内打开名片并浏览公开内容。
- [ ] 访客在提示后可与 AI 多轮对话；AI 明确身份。
- [ ] 有依据回答可在后台追溯；无依据、敏感和越权问题正确拒答。
- [ ] 访客明确同意后可留资；主人可查看并跟进线索。
- [ ] 会话可幂等生成结构化拜访纪要。
- [ ] 知识缺口可审核，批准后生成新版本并重新索引。
- [ ] 站内可查看新线索、纪要和索引失败；外部消息不是 MVP 阻塞项，除非甲方另行冻结。

### 10.2 AI 质量

- [ ] 每家验收企业至少 20 题，数据集和答案来源版本固定。
- [ ] Retrieval Hit@5 ≥ 85%，有资料回答通过率 ≥ 80%，无资料拒答率 ≥ 90%。
- [ ] 事实引用支持率 100%；不存在虚构客户、资质、报价或合同承诺。
- [ ] 跨企业泄露和严重 Prompt injection 失败为 0。
- [ ] 纪要 Schema 完整率 100%，抽样事实均能回溯到对话。

### 10.3 安全、合规与运维

- [ ] A/B 企业的 API、数据库、向量、对象存储、缓存路径均完成越权测试。
- [ ] PII 加密、列表脱敏、敏感查看审计；日志和错误响应无秘密/明文 PII。
- [ ] 隐私政策、AI 标识、同意文案、数据主体请求流程经责任人批准。
- [ ] 关键接口限流；Token 撤销、IDOR、CSRF/CORS、输入长度测试通过。
- [ ] 备份成功且完成一次恢复演练，RPO/RTO 达标。
- [ ] 模型/Redis/对象存储故障降级通过；没有静默丢失的异步任务。
- [ ] 监控覆盖错误率、延迟、队列、数据库、模型用量/成本和索引失败。

### 10.4 性能

- [ ] 名片首屏 P75 ≤ 2.5 秒（约定 4G/设备/生产构建）。
- [ ] AI 首 token P95 ≤ 5 秒、完整回答 P95 ≤ 10 秒（约定模型和并发）。
- [ ] 常用后台列表 P95 ≤ 1 秒。
- [ ] 目标试点并发与 2 倍短时峰值压测通过，容量参数写入报告。

## 11. 缺陷与发布规则

- P0：数据泄露、无法登录/打开名片、核心闭环中断、AI 重大虚构/违法输出、数据不可恢复。发布阻断。
- P1：主要功能错误、有替代路径但影响试点、明显性能不达标。原则上发布阻断；例外需产品/技术/测试共同书面接受。
- P2：非核心体验或样式问题，可带明确版本计划发布。

生产发布采用迁移前备份、向后兼容 migration、健康检查、冒烟、逐步放量和可执行回滚。数据库破坏性变更使用 expand/contract，不依赖“回滚镜像即可恢复数据库”。

## 12. 项目风险看板

| 风险 | 早期信号 | 负责人 | 处理 |
|---|---|---|---|
| 样板资料不足 | 20 FAQ/案例不能按时提供 | 产品/甲方 | 先完成资料模板；AI 验收不启动 |
| 范围膨胀 | Sprint 中新增外部渠道/数字人 | 产品 | 走变更控制，交换等量范围 |
| 模型质量不稳 | 固定集波动或拒答不足 | AI | 固定版本、Prompt 回归、降级/切换 |
| 多租户泄露 | 发现无 scope 的 Repository/查询 | 技术/安全 | RLS + 自动化阻断合并 |
| 试点资源未落实 | 无名单、无运营负责人 | 业务负责人 | 不进入 V1.5，不以推广量验收研发 |
| 外部平台受限 | 公众号/企微主体和授权不全 | 产品/集成 | 站内通知先行，Provider 后置 |
| 成本失控 | 单会话 token/重试持续上升 | AI/运维 | 预算、上下文限制、缓存、告警 |
