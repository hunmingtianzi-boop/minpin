## ADDED Requirements

### Requirement: 企业工作区保持角色安全的信息架构
系统 SHALL 为企业管理员和名片主人提供业务概览、客户旅程、知识运营、内容资产和企业治理分组，并根据后端权限隐藏和拒绝无权功能。

#### Scenario: 企业管理员登录
- **WHEN** 企业管理员登录
- **THEN** 系统进入企业工作区并展示其企业范围内允许的导航和数据

#### Scenario: 名片主人登录
- **WHEN** 名片主人登录
- **THEN** 系统只展示本人允许的名片、访问、对话、线索和纪要范围

### Requirement: 企业概览展示真实业务闭环
企业概览 SHALL 使用当前 API 的真实指标展示访问、有效对话、线索、活跃名片、知识状态和待处理事项，并为每项提供清晰入口。

#### Scenario: 待处理资料存在
- **WHEN** 企业有导入草稿、知识缺口或失败任务
- **THEN** 概览显示数量、状态和进入处理页面的动作

### Requirement: 企业名片发布入口与员工名片管理分离
企业控制台 SHALL 将“企业名片”和“员工名片”作为两个清晰业务区域。企业名片区域 MUST 提供企业官方名片的新建/编辑、公开预览、发布和下线；员工名片区域 MUST 保留按员工所有者管理个人名片的能力。两种类型不得仅靠展示名称推断。

#### Scenario: 企业尚无官方名片
- **WHEN** 企业管理员进入企业名片区域且不存在企业官方名片
- **THEN** 系统显示创建企业名片的明确主动作，而不是要求先选择员工

#### Scenario: 企业官方名片待发布
- **WHEN** 企业官方名片处于草稿状态
- **THEN** 企业管理员可编辑并显式发布，发布后使用服务端 `share_url` 打开真实公开页

### Requirement: 控制台状态和响应式体验一致
企业控制台关键页面 MUST 覆盖 loading、empty、error、success 和权限拒绝状态；桌面与 390px 窄屏均不得隐藏主要动作。

#### Scenario: API 请求失败
- **WHEN** 企业页面 API 请求失败
- **THEN** 页面显示脱敏错误、请求 ID 和重试动作，而不是空白或伪成功

#### Scenario: 窄屏名片管理
- **WHEN** 用户在 390px 视口管理名片
- **THEN** 名片状态、编辑、分享和公开预览动作均可见且可操作

### Requirement: 平台与企业视觉身份可区分但共享组件语言
平台工作区 SHALL 使用克制琥珀识别色，企业工作区 SHALL 使用现有蓝色识别；两者共享当前 Fluent、8px 圆角、Panel、Table、StatusBadge、焦点与错误反馈规则，不引入渐变或玻璃拟态。

#### Scenario: 跨角色视觉识别
- **WHEN** 用户分别进入平台和企业工作区
- **THEN** 页面身份、导航分组和强调色清晰不同，但组件密度和交互规则一致
