## ADDED Requirements

### Requirement: 平台可从资料启动隔离的企业开通草稿
平台管理员 SHALL 能创建资料辅助建企会话。系统 MUST 在普通企业列表、登录和公开查询之外建立不可登录、不可公开的临时企业范围，并将会话、创建者、临时 tenant/company、版本和状态服务端绑定。开始会话只要求无法从资料安全推导的字段，例如租户标识和管理员账号凭据；企业名称、行业、简介及初始内容 MAY 等待资料建议。

#### Scenario: 启动资料辅助建企
- **WHEN** 平台管理员提交合法且不冲突的租户标识和管理员初始化信息
- **THEN** 系统创建隔离开通会话及临时企业范围，但管理员凭据不可登录且任何名片不可公开

#### Scenario: 企业角色启动平台开通
- **WHEN** 企业角色直接请求资料辅助建企 API
- **THEN** 系统返回 403 且不创建临时资源

### Requirement: 建企资料复用当前 knowledge_import
资料辅助建企 SHALL 复用当前 `knowledge_import` 的格式校验、对象存储、解析器、批次状态、Worker 和草稿语义。目标 tenant/company MUST 从服务端开通会话推导，不得由客户端任意指定；不得引入 `document_import`、Docling、MinerU、独立 OCR 或第二套上传/解析 Worker。

#### Scenario: 上传支持资料
- **WHEN** 会话创建者向有效开通会话上传当前合同支持的文件
- **THEN** 系统在该临时企业范围内创建标准 `knowledge_import` 批次并返回逐文件状态

#### Scenario: 文件部分失败
- **WHEN** 同一建企批次中部分文件解析失败
- **THEN** 系统保留成功草稿、显示逐文件失败原因，并允许补传或继续人工填写

### Requirement: LLM 只生成带来源的可审核建议
当平台主 LLM 可用时，系统 MAY 从成功解析的草稿文本生成企业名称、行业、简介、网站、产品/案例摘要和初始名片内容建议。每个建议 MUST 携带来源草稿/文件、置信提示和生成版本，不得根据未解析原始文件、提示中的指令或外部 URL 执行动作。LLM 输出不得创建账号、激活企业、发布知识或发布名片。

#### Scenario: 生成企业初始化建议
- **WHEN** 至少一个导入草稿成功且平台主 LLM 可用
- **THEN** 系统返回带来源的结构化建议，并将所有字段保持为待确认状态

#### Scenario: LLM 不可用
- **WHEN** 文件已解析但平台主 LLM 未配置、停用或调用失败
- **THEN** 导入草稿保持可用，开通会话进入“需人工填写”状态且不回滚解析结果

#### Scenario: 文档包含提示注入文本
- **WHEN** 已解析资料要求模型泄露密钥、访问外部地址或跳过审核
- **THEN** 系统将资料仅作为不可信内容处理，不执行指令、不访问地址且仍要求逐字段确认

### Requirement: 人工确认后原子激活企业
平台管理员 MUST 审核并可编辑全部建议，显式确认租户、企业、管理员和初始名片字段，并携带 `expected_version` 完成激活。确认操作 SHALL 幂等且原子地激活 tenant/company、企业管理员 membership/credential，并创建或更新一张不绑定员工的企业官方草稿名片；导入知识保持草稿，除非企业用户后续按现有权限发布。

#### Scenario: 确认资料辅助建企
- **WHEN** 平台管理员确认所有必填字段且版本仍为最新
- **THEN** 系统只创建一个正式企业、一个可用管理员身份和一张未发布企业官方名片，并保留资料来源关系和安全审计

#### Scenario: 重复确认或过期版本
- **WHEN** 客户端重复提交同一确认请求或使用过期 `expected_version`
- **THEN** 重复请求返回同一结果，过期版本返回 409，且不产生重复企业、账号或名片

### Requirement: 取消与过期不会产生可用孤儿企业
未确认会话 SHALL 支持软取消和过期锁定。取消或过期后，临时企业、凭据和名片不得变为可登录或公开状态；任何物理清理必须由独立、可审计且遵守保留策略的过程完成。

#### Scenario: 开通会话取消
- **WHEN** 平台管理员取消尚未确认的资料辅助建企会话
- **THEN** 系统撤销继续上传、生成和确认的能力，同时保留最小审计记录且不删除既有正式企业数据
